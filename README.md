# افتح يا سمسم — لعبة الأسئلة | Open Sesame Trivia

**🎮 العب الآن: [iftah-ya-simsim.vercel.app](https://iftah-ya-simsim.vercel.app)**

يشتغل مباشرة في المتصفح — بدون تثبيت. على الجوال: افتح الرابط بسفاري ثم
«إضافة إلى الشاشة الرئيسية» فيصير أيقونة تفتح بملء الشاشة مثل أي تطبيق.

لعبة معلومات ثنائية اللغة (عربي / إنجليزي) بنظام لعب على طريقة **كاهوت**، وبتصميم مستوحى من
مسلسل «افتح يا سمسم». مبنية بـ React Native + Expo، وتعمل مباشرة في **Expo Go** بدون أي بناء أصلي.

A bilingual (Arabic / English) trivia game with **Kahoot-style** gameplay and an
*Open Sesame* / Sesame Street visual design. Built with React Native + Expo, runs directly in **Expo Go**.

## المتطلبات | Requirements

| | الإصدار |
|---|---|
| Expo SDK | **54** |
| React Native | 0.81.5 |
| React | 19.1.0 |

⚠️ **مهم:** تطبيق **Expo Go** على جوالك لازم يدعم **SDK 54**.

للتأكد: افتح Expo Go → **Settings** → قسم **App Info** → سطر **Supported SDK**.
لو الرقم غير ٥٤، إما تحدّث/تنزّل النسخة المناسبة من [expo.dev/go](https://expo.dev/go)،
أو تضبط إصدارات المشروع على الرقم اللي يدعمه تطبيقك.

رسائل الخطأ الشائعة:
- *"requires a **newer** version of Expo Go"* → تطبيقك أقدم من المشروع
- *"requires an **older** version"* → تطبيقك أحدث من المشروع

## التشغيل | Running

```bash
npm install
npx expo start
```

افتح تطبيق **Expo Go** على جوالك وامسح رمز الـ QR الظاهر في الطرفية.
الجوال والكمبيوتر لازم على نفس شبكة الواي فاي — وإذا ما اشتغل: `npx expo start --tunnel`

للتجربة في المتصفح: `npm run web`

## الباركود | QR code

لتوليد باركود جاهز للمشاركة (صورة + صفحة عرض):

```bash
npm run qr
```

يطبع الباركود في الطرفية ويحفظ:

| الملف | الاستخدام |
|---|---|
| `qr.png` | صورة ترسلها لأي أحد ليمسحها |
| `qr.html` | صفحة كاملة للعرض على الشاشة أو الطباعة |

الباركود يشير إلى `exp://<عنوانك>:8081`. **لازم يكون `npx expo start` شغّالاً** والجوال على نفس الواي فاي.

إذا كنتم على شبكات مختلفة، استعمل النفق ثم مرّر رابطه:

```bash
npx expo start --tunnel
npm run qr -- exp://xxxx.exp.direct
```

## اللغتان | Languages

- **شاشة اختيار اللغة تظهر أول ما يفتح التطبيق** — عربي أو English، والنصوص فيها باللغتين.
- بعدها زر 🌐 في أعلى الشاشة الرئيسية يبدّل بينهما في أي وقت.
- كل الواجهة وكل سؤال مكتوبان باللغتين — ليست ترجمة آلية.
- العربية تُعرض من اليمين لليسار (RTL) دون الحاجة لإعادة تشغيل التطبيق.

## المباراة والفريقان | Match & teams

**وضعان:**

| الوضع | الوصف |
|---|---|
| 👤 لاعب واحد | تلعب وحدك، وتظهر لك نجوم ونسبة دقة ورقم قياسي |
| 👥 فريقان | تسمّي الفريقين وتتناوبون على نفس الجهاز |

**في وضع الفريقين:**
- الفريقان يتناوبان **سؤالاً بسؤال** — سؤال للأحمر، سؤال للأزرق، وهكذا.
- شاشة «استعد» تعلن دور من، ولون الشاشة كله يتغيّر بلون الفريق صاحب الدور.
- نقاط كل فريق ظاهرة دائماً في الأعلى، وصندوق الفريق صاحب الدور مضيء والآخر باهت.
- في النهاية يُعلَن **الفائز** 🏆 مع نقاط الفريقين وعدد إجاباتهما الصحيحة وأطول متتالية، أو **تعادل** إذا تساوت النقاط.

**نظام الجولات** — قابل للضبط قبل كل مباراة:

| الإعداد | الخيارات | الافتراضي |
|---|---|---|
| 🎯 عدد الجولات | ٢ / ٣ / ٥ | ٣ |
| ❓ أسئلة كل جولة | ٤ / ٦ / ٨ | ٦ |
| ⏱️ وقت السؤال | ١٠ / ٢٠ / ٣٠ ثانية | ٢٠ |

بين كل جولة وأخرى تظهر **شاشة ترتيب** توضّح نقاط كل فريق ومن المتقدّم، وبعد آخر جولة تنتهي المباراة بشاشة النتيجة النهائية.

## نظام اللعب (على طريقة كاهوت) | Kahoot-style gameplay

| العنصر | التفصيل |
|---|---|
| العدّ التنازلي | ٣ ← ٢ ← ١ قبل كل سؤال مع «استعد!» ودور الفريق |
| المؤقّت | وقت محدّد لكل سؤال، شريط يتقلّص ولونه يتغيّر (أخضر ← أصفر ← أحمر) |
| الخيارات | أربع بلاطات، لكل واحدة **لون + شكل** خاص: 🔺 أحمر، 🔷 أزرق، ⭕ أصفر، ⬛ أخضر |
| النقاط | كلما أسرعت زادت النقاط: من ٥٠٠ (عند آخر ثانية) إلى **١٠٠٠** (إجابة فورية) |
| المتتالية | إجابتان صحيحتان متتاليتان = +١٠٠، وثلاث فأكثر = +٢٠٠ 🔥 |
| الكشف | بعد كل إجابة: صح/خطأ + النقاط المكتسبة + معلومة إضافية 💡 |
| بين الجولات | شاشة ترتيب: نقاط كل فريق، إجاباته الصحيحة، أطول متتالية |
| النهاية | الفائز 🏆 (فريقان) أو ٣ نجوم ودقة ورقم قياسي (لاعب واحد) + قصاصات احتفال |

## المؤثرات الصوتية | Sound effects

٩ مؤثرات **مولّدة برمجياً** داخل المشروع (نغمات مركّبة بصيغة WAV، ١٦٥ كيلوبايت إجمالاً) —
لا شيء يُحمَّل من الإنترنت ولا مكتبات صوت خارجية.

| الصوت | متى يشتغل |
|---|---|
| `tap` | نقر أي زر أو بطاقة (+ اهتزاز خفيف) |
| `count` | كل رقم في العدّ التنازلي ٣ ← ٢ ← ١ |
| `go` | انطلاق السؤال |
| `tick` | دقّات تحذيرية في الثواني الخمس الأخيرة |
| `correct` | إجابة صحيحة — أربع نغمات صاعدة (+ اهتزاز نجاح) |
| `wrong` | إجابة خاطئة — نغمتان هابطتان (+ اهتزاز خطأ) |
| `timeup` | انتهاء الوقت — جرس إنذار |
| `streak` | ثلاث إجابات صحيحة متتالية فأكثر 🔥 |
| `finish` | نهاية الجولة بنجمتين فأكثر — فانفير |

## الموسيقى الخلفية | Background music

مقطوعتان خفيفتان تدوران بلا توقف، مولّدتان برمجياً أيضاً:

| المقطوعة | متى | الوصف |
|---|---|---|
| `bg_menu` | القوائم وشاشة النتيجة | هادئة وبطيئة (≈٧٦ نبضة)، ١٨ ثانية |
| `bg_game` | أثناء المباراة | مرحة وخفيفة (≈١٠٨ نبضة) مع باص وشيكر، ٢٥ ثانية |

- **تكرار بلا فاصل**: طول كل لفة يقبل القسمة على الإيقاع بالضبط، وذيول النغمات
  تلتفّ إلى بداية اللفة — فتكون الموسيقى دورية رياضياً بلا أي طقطقة عند التكرار.
- **مستوى منخفض** (٤٢٪) حتى لا تزاحم المؤثرات الصوتية.
- **تخفيض تلقائي**: تنخفض الموسيقى إلى الربع لمدة ٢٫٤ ثانية عند فانفير النهاية ثم تعود.

## التحكّم بالصوت | Audio controls

ثلاثة أزرار في أعلى الشاشة الرئيسية:

| الزر | الوظيفة |
|---|---|
| 🔊 / 🔇 | المفتاح الرئيسي — يكتم المؤثرات والموسيقى والاهتزاز معاً |
| 🎵 | الموسيقى وحدها (يظهر عليها خط مائل عند الإغلاق) |
| ؟ | شرح اللعبة |

- الصوت يشتغل حتى لو الجوال على الوضع الصامت. لتغيير ذلك، عدّل
  `playsInSilentMode` إلى `false` في [src/sound.js](src/sound.js).
- مستوى الموسيقى في `MUSIC_VOLUME` في نفس الملف.

## توليد الصوتيات | Regenerating audio

كل الملفات الصوتية مولّدة بسكربتات داخل المشروع — تقدر تعدّل النغمات وتعيد توليدها:

```bash
npm run gen:sfx      # المؤثرات التسعة
npm run gen:music    # المقطوعتان الخلفيتان
npm run gen:audio    # الاثنان معاً
```

السلالم والكوردات والإيقاع كلها في `scripts/gen-music.js` و`scripts/gen-sounds.js`.
سكربت الموسيقى يفحص وصلة التكرار تلقائياً ويطبع ✅ أو ⚠️.

### منع التكرار | No repeats

اللعبة تتذكّر كل سؤال طُرح عليك، ولا تعيده حتى تُنهي بنك الفئة والمستوى بالكامل.
الرقم الصغير على كل بطاقة فئة/مستوى يوضّح كم سؤالاً جديداً بقي لك.
عند انتهاء البنك تُعاد التهيئة تلقائياً مع رسالة تهنئة.

## التصميم | Design

كل الرسومات مبنية بعناصر `View` و`Animated` — **لا توجد أي صور ولا خطوط خارجية**:

- **خمس دُمى** بستايل سمسم: الطائر 🐤، الوحش الفروي، النكد في صندوقه، الجمل 🐪،
  ووحش العيون المتدحرجة — لكل واحدة عيون بارزة ترمش، وترنّح مستمر، وتعابير (فرح / حزن / تفكير).
- **مشهد الشارع**: عمارة طوب بلوحة «١٢٣»، درج، شباك، فانوس، رصيف، شمس تدور بأشعتها، وسحب تمشي.
- **أزرار الجوخ**: حافة سفلية سميكة تنضغط للأسفل عند اللمس.
- ألوان المسلسل: أصفر شمسي، أحمر، أزرق، أخضر، بنفسجي، وطوب برتقالي.

## إضافة أسئلة | Adding questions

افتح أي ملف في `src/data/` وأضف عنصراً إلى `easy` أو `medium` أو `hard`:

```js
{
  a: 2,  // رقم الإجابة الصحيحة، تبدأ من صفر
  ar: { q: 'نص السؤال؟', o: ['خيار', 'خيار', 'الصحيح', 'خيار'], f: 'معلومة بعد الإجابة.' },
  en: { q: 'The question?', o: ['opt', 'opt', 'correct', 'opt'], f: 'A fact shown after answering.' },
}
```

الخيارات تُخلط تلقائياً في كل جولة، وبنفس الترتيب في اللغتين.

## ملاحظات | Notes

- خيارات الجولات والوقت والنقاط في `src/theme.js`
  (`ROUND_OPTIONS`, `PER_ROUND_OPTIONS`, `TIME_OPTIONS`, `DEFAULT_SETUP`, `MAX_POINTS`).
- ألوان الفريقين وشخصياتهما في `TEAM_STYLES` في نفس الملف.
- أفضل نتيجة والأسئلة المطروحة تُحفظ أثناء تشغيل التطبيق فقط، وتُصفَّر عند إغلاقه.
- `react-dom` و`react-native-web` مضافة للتجربة في المتصفح فقط، ولا تؤثر على Expo Go.
